# Results Directory

This directory contains all outputs generated by the benchmark experiments for the **Travelling Salesman Problem**.

All files are automatically produced by the analysis pipeline after running the experiments.

---

# Files Description

## experiment_results.csv

Raw experimental data generated by `run_experiments.py`.

Each row represents a solver execution for a specific problem instance.

Columns description:

- **instance** → name of the benchmark instance  
- **solver** → solver used in the experiment  
- **n_items** → number of items in the knapsack instance  
- **capacity** → knapsack capacity  
- **objective** → objective value found by the solver  
- **known_optimal** → known optimal solution value for the instance  
- **gap** → difference between optimal solution and solver solution  
- **found_optimal** → flag indicating whether the solver reached the optimal solution (1 = yes, 0 = no)  
- **runtime** → execution time in seconds  
- **status** → solver status (Optimal, Not Solved, etc.)

This dataset is the primary source for all statistical analysis and visualizations.

---

## performance_statistics.csv

Aggregated performance metrics computed from `experiment_results.csv`.

Statistics are grouped by solver and include:

- **mean_runtime** → average execution time
- **median_runtime** → median execution time
- **std_runtime** → runtime variability
- **mean_objective** → average objective value obtained
- **median_objective** → median objective value
- **mean_gap** → average absolute optimality gap
- **median_gap** → median gap
- **mean_relative_gap** → average normalized optimality gap
- **optimal_rate** → percentage of optimal solutions found
- **timeout_rate** → percentage of runs that were not solved

This file summarizes the overall solver performance across all benchmark instances.

---

## runtime_boxplot.png

Boxplot visualization of runtime distribution per solver.

This plot highlights:

- Median runtime
- Runtime variability
- Outliers
- Comparative performance between solvers

It is used to visually assess computational efficiency.

---

## objective_boxplot.png

Boxplot showing the distribution of objective values found by each solver.

This plot helps identify:

- Differences in solution quality
- Consistency of solutions across instances
- Possible deviations from the optimal solution

Ideally, both solvers should show very similar distributions.

---

## gap_boxplot.png

Boxplot of the relative optimality gap per solver.

The relative gap is computed as:

```bash
relative_gap = (known_optimal - objective) / known_optimal
```


This visualization highlights:

- How close each solver is to the optimal solution
- Stability of solution quality
- Instances where near-optimal solutions occur

A distribution concentrated near zero indicates high solution quality.

---

## statistical_test.txt

Results of the statistical significance test comparing solver runtimes.

The type of test depends on the number of solvers evaluated:

- **≥ 3 solvers** → Friedman Test
- **2 solvers** → Wilcoxon Signed-Rank Test

The file reports:

- Test name
- Solvers compared
- Test statistic
- p-value

This analysis determines whether runtime differences between solvers are statistically significant.

---

## success_rate.png

Bar chart showing the optimal solution detection rate for each solver.

This metric represents the percentage of instances where the solver found the known optimal solution.

It provides a clear comparison of solver reliability.

---

# How These Results Are Generated

All files in this directory are automatically generated by running:

```bash
python -m analysis.analyze_results
```

The analysis script performs the following steps:

Loads experimental results
Computes relative optimality gaps
Generates aggregated statistics
Creates visualizations
Runs statistical significance tests

The results are fully reproducible.
